# IntelliExtract 方法论

## 项目概述

IntelliExtract 是一个基于浏览器的 AI 数据摘录工具，通过上传 PDF 文档与 Excel 需求清单，利用大语言模型自动提取结构化数据。

---

## 1. 系统架构

### 1.1 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 构建工具 | Vite | 支持 `.env` 环境变量、ES Module、热更新 |
| 前端框架 | 原生 HTML/CSS/JS | 无额外框架依赖 |
| PDF 解析 | pdf.js (CDN) | 仅 OpenAI 兼容模式下使用本地文本解析 |
| Excel 读写 | SheetJS (xlsx) | npm 包，支持读写 `.xlsx`，保留 leading zero |
| AI 接口 | Google Gemini / OpenAI 兼容 | 自动检测 API 类型 |

### 1.2 前端架构

```
纯客户端架构（无后端服务器）
┌──────────────┐
│   浏览器      │
│  ┌─────────┐ │     ┌──────────────────┐
│  │ PDF文件  │──────▶│ Gemini: 原生PDF   │
│  │ Excel文件│──────▶│ OpenAI: 文本解析  │
│  └─────────┘ │     └────────┬─────────┘
│              │              │
│  ┌─────────┐ │              ▼
│  │ 结果表格 │◀──── JSON 结构化返回
│  │ Excel导出│ │
│  └─────────┘ │
└──────────────┘
```

**隐私优势**：所有文件解析均在浏览器本地完成，数据仅通过加密链路直达 AI 官方接口。

## 2. 输入规范

### 2.1 PDF 文件

- 格式：`.pdf`
- 处理方式：
  - **Gemini 模式**：PDF 转 Base64 后直接发送给模型（原生文档理解）
  - **OpenAI 模式**：使用 pdf.js 本地逐页提取文本，附带 `[Page X]` 标记

### 2.2 Excel 需求清单

- 格式：`.xlsx` / `.xls`
- 必须包含以下列：

| 列名 | 类型 | 说明 |
|------|------|------|
| `indicator_code` | 字符串 | 指标代码（支持 leading zero） |
| `indicator_name` | 字符串 | 指标名称 |
| `indicator_type` | 字符串 | `文本型` / `数值型` / `强度型` |
| `definition` | 字符串 | 指标定义 |
| `guidance` | 字符串（可选） | 补充指引 |

---

## 3. 输出规范

### 3.1 结果表格字段

| 列名 | 适用类型 | 说明 |
|------|---------|------|
| `indicator_code` | 全部 | 指标代码 |
| `indicator_name` | 全部 | 指标名称 |
| `indicator_type` | 全部 | 文本型 / 数值型 / 强度型 |
| `year` | 全部 | 年份（多年数据自动拆分为多行） |
| `text_value` | 文本型 | 提取的文本结果 |
| `num_value` | 数值型 / 强度型 | 提取的数值 |
| `unit` | 数值型 / 强度型 | 数值的单位 |
| `page_number` | 全部 | PDF 的**物理页码**（非内容印刷页码） |

### 3.2 导出规则

- 格式：`.xlsx`
- `indicator_code` 列强制为文本格式（保留 leading zero）
- **未找到结果的指标不包含在导出文件中**

---

## 4. 批处理策略

### 4.1 分类分批

指标按 `indicator_type` 分为两组，各组独立分批：

| 组别 | 包含类型 | 每批上限 | 原因 |
|------|---------|---------|------|
| 文本组 | `文本型` | 100 个 | 文本返回较长，需控制输出 token |
| 数值组 | `数值型` + `强度型` | 200 个 | 数值返回简短，可提高吞吐 |

**每个批次仅包含同一类型的指标**，确保 System Prompt 精准无歧义。

### 4.2 多年份处理

如果同一指标在 PDF 中包含多个年份的数据，模型会为每个年份返回**独立的结果行**，通过 `year` 字段区分。

## 5. API 集成

### 5.1 双模式支持

| 模式 | 检测方式 | 认证方式 | PDF 处理 |
|------|---------|---------|---------|
| Google Gemini | URL 含 `googleapis.com` | `x-goog-api-key` Header | 原生 Base64 传输 |
| OpenAI 兼容 | 其他 URL | `Authorization: Bearer` | 本地文本解析 |

### 5.2 重试机制

- 最多重试 **3 次**
- 指数退避：1s → 2s → 4s（上限 16s）
- 每次重试在进度日志中记录错误信息

### 5.3 安全措施

- API Key 存储在 `.env` 文件中（`VITE_GEMINI_API_KEY`）
- `.env` 被 `.gitignore` 排除，不会进入版本控制
- API Key 不出现在 URL 参数中，仅通过 HTTP Header 传输
- 配置面板在 UI 中隐藏

---

## 6. 前端交互

### 6.1 文件上传

- 支持点击选择和**拖拽上传**
- 拖拽时有视觉高亮反馈
- 文件类型校验：同时检查 MIME 类型和文件扩展名

### 6.2 实时进度

- 每完成一个批次，结果表格**立即更新**（无需等待全部完成）
- 进度条和百分比实时刷新
- 详细日志面板记录每个步骤的时间戳

### 6.3 结果过滤

- 提供 "Show only results found" 复选框
- 勾选后隐藏未找到结果的行（文本型检查 `text_value`，数值型检查 `num_value`）

```bash
# 开发模式（热更新）
npm run dev

# 生产构建
npm run build

# 预览构建产物
npm run preview
```

```
VITE_GEMINI_API_KEY=your_api_key_here
```
