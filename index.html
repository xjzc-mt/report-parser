IntelliExtract

AI-Powered Data Extraction from PDFs via Excel Requirements

📖 Methodology / 方法论 ▶

1. 系统架构

纯客户端架构 — 所有文件解析均在浏览器本地完成，数据仅通过加密链路直达 AI 官方接口。

层级技术说明
构建工具Vite支持 .env 环境变量、ES Module
PDF 解析pdf.js仅 OpenAI 模式下使用本地文本解析
Excel 读写SheetJS保留 leading zero
AI 接口Gemini / OpenAI自动检测 API 类型

2. 输入规范

PDF 文件

  • Gemini 模式：PDF 转 Base64 后直接发送（原生文档理解）
  • OpenAI 模式：pdf.js 本地逐页提取文本，附带 [Page X] 标记

Excel 需求清单 (.xlsx)

列名类型说明
indicator_code字符串指标代码（支持 leading zero）
indicator_name字符串指标名称
indicator_type字符串文本型/文字型 / 数值型 / 强度型
definition字符串指标定义
guidance可选补充指引

3. 输出规范

列名适用类型说明
indicator_code全部指标代码
indicator_name全部指标名称
indicator_type全部文本型 / 数值型 / 强度型
year全部年份（多年份数据自动拆分为多行）
text_value文本型提取的文本结果
num_value数值 / 强度提取的数值
unit数值 / 强度数值的单位
page_number全部PDF 物理页码（非内容页码）

导出规则：未找到结果的指标不包含在输出文件中。

4. 批处理策略

组别包含类型每批上限原因
文本组文本型100 个文本返回较长
数值组数值型 + 强度型200 个数值返回简短
  • 每个批次仅含同一类型的指标
  • 多年份数据自动拆分为独立行
  • 失败批次最多重试 3 次（指数退避：1s → 2s → 4s）

5. API 集成

模式检测方式认证PDF 处理
GeminiURL 含 googleapis.comx-goog-api-key Header原生 Base64
OpenAI其他 URLAuthorization: Bearer本地文本解析

6. 安全措施

  • API Key 存储在 .env 文件中，被 .gitignore 排除
  • Key 不出现在 URL 参数中，仅通过 HTTP Header 传输
  • 配置面板在 UI 中隐藏
